Port 80 Binding Instructions

If you want to run WordPress Multisite directly on port 80 (e.g. access as http://wp.localhost without appending a port number), you may need to authorize your system to bind to privileged ports:

Linux (Rootless Podman / Docker)
Rootless containers cannot bind to ports below 1024 by default. Run the following command in your terminal to allow binding to port 80: 
sudo sysctl -w net.ipv4.ip_unprivileged_port_start=80
To make this setting persistent across reboots: 
echo "net.ipv4.ip_unprivileged_port_start=80" | sudo tee -a /etc/sysctl.d/99-podman-ports.conf
Windows
Docker Desktop and WSL2 automatically handle port forwarding. If port 80 is blocked by IIS or another system process, disable it in an Administrator command prompt: 
net stop http
macOS
Docker Desktop maps port 80 automatically. Ensure the built-in macOS Apache server is stopped: 
sudo apachectl stop
